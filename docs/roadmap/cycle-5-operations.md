# 周期 5:运维与弹性

**状态**:未开始

## 北极星目标

集群可长期承载真实工作负载:节点能计划性维护、虚机能跨节点迁移、磁盘能在线快照与备份、关键指标可被采集、关键事件可被告警、控制面重启不丢状态。

## 承诺

本周期完成后:

- 节点维护窗口期间业务不中断:通过排空 + 迁移完成。
- 单台节点失联不影响其它节点上虚机的服务。
- 关键虚机可做在线快照,并可基于快照点恢复出新虚机。
- 关键虚机可备份导出,在另一节点恢复。
- 控制面与节点提供 Prometheus 风格的 metrics 端点。
- 控制面进程重启后状态完整恢复,无虚机状态错位。

## 范围内

- 在线热迁移:基于 QMP migrate,源/目标 govirtlet 由控制面建立链路并协调状态。
- 在线快照:走 QMP blockdev-snapshot-sync 与 block-commit;离线快照仍走 qemu-img。
- 备份导出:qemu-img convert 把 overlay 合并成可搬运的 qcow2 / raw。
- 节点排空:节点标记 unschedulable,其上虚机被迁出,直至空载。
- 调度策略增强:节点亲和、反亲和、污点与容忍。
- 指标暴露:govirtad 与 govirtlet 各自 `/metrics`,字段稳定。
- 控制面持久化加固:存储后端可选择经过验证的本地数据库,重启后状态完整恢复。

## 范围外

- 跨数据中心异地容灾。
- 跨集群联邦。
- 自动扩缩与成本调度。
- 网络叠加层(VXLAN、SDN)。
- 以上事项进入 v2 规划,不在本路线图。

## 完成判定

- [ ] 一台运行业务的虚机可在不停机状态下从节点 A 迁移到节点 B,迁移期间业务可继续。
- [ ] 一台虚机可触发在线快照,并能从该快照恢复出新虚机,数据一致。
- [ ] 一台虚机可被备份导出为独立 qcow2 文件,在另一节点导入并启动成功。
- [ ] 节点 A 标记 drain 后,其上所有虚机在阈值内迁出,节点变为空载。
- [ ] 调度器支持亲和、反亲和、污点容忍,并有集成测试覆盖。
- [ ] govirtad 与 govirtlet 提供 Prometheus 兼容 metrics 端点,字段命名稳定。
- [ ] 控制面进程重启后所有 Node 与 VirtualMachine 状态完整恢复,无错位。
- [ ] 故障注入(模拟节点宕机)下集群其它部分继续可用,虚机状态被正确标记。
- [ ] `go test ./...` 与 `go test -race ./...` 全部通过。
- [ ] 本文件状态切换到「已完成」并附验收证据。
