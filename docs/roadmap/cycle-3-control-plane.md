# 周期 3:控制面与调度

**状态**:未开始

## 北极星目标

多台 govirtlet 节点 + 单一 govirtad 控制面 + govirtctl 远程使用,从 CLI 提交一台虚机,能被调度并真实运行在某台节点上;期望状态与实际状态通过控制循环持续收敛。

## 承诺

本周期完成后:

- govirtad 是整集群唯一的 API 入口,govirtctl 通过它操作所有虚机。
- 多个 govirtlet 节点能向控制面注册并周期心跳,控制面能识别上线/掉线。
- 调度器能根据基本资源(CPU、内存、节点在线状态)选择目标节点。
- 控制平面写入 spec → 节点执行 → 节点回写 status 的闭环跑通。

## 范围内

- govirtad apiserver:统一路由,资源对象基于 spec / status 分离。
- 资源建模:Node、VirtualMachine 两个核心对象。
- 节点注册与心跳:govirtlet 启动后向 govirtad 注册并维持租约,失联后控制面在阈值内识别。
- 调度器:最小过滤 + 打分接口,首版覆盖资源容量与节点在线状态两个维度。
- 控制循环:apiserver 写入 spec → 控制器 diff 计算 → 下发到节点 → 节点回写 status。
- govirtctl 远程模式:`govirtctl --server <addr> vm create/list/...` 全部走控制面。

## 范围外

- 多副本 govirtad、强一致存储、HA → 不在本周期。
- 模板/镜像服务化 → 周期 4。
- 完整 RBAC 矩阵 → 周期 4 起步,周期 5 加强。
- 跨节点网络叠加 → 假设各节点共享 L2 网络或外部已规划。
- 热迁移、在线快照、监控指标 → 周期 5。

## 完成判定

- [ ] govirtad 暴露稳定 API,能 CRUD Node 与 VirtualMachine 两个对象。
- [ ] govirtlet 启动后自动注册,租约/心跳机制工作正常,失联节点在阈值内被识别。
- [ ] 调度器能在 ≥2 节点中根据资源选择目标并下发任务。
- [ ] `govirtctl vm create` 通过控制面创建虚机,虚机最终在被选中的节点上启动。
- [ ] 节点上的虚机实际终止后,控制面 status 在阈值内被推到对应终态。
- [ ] 集成测试覆盖:节点注册 → 创建 → 调度 → 启动 → 状态回流 → 删除。
- [ ] `go test ./...` 与 `go test -race ./...` 全部通过。
- [ ] 本文件状态切换到「已完成」并附验收证据。

## 进入下一周期的前置

- 资源对象 spec / status 模型已稳定:周期 4 在其上扩展业务字段(VNC、cloud-init、模板)。
- 调度器接口已抽象:周期 5 在其上加亲和、反亲和、节点排空等高级策略。
- 控制循环模式已验证:周期 5 的迁移与快照编排沿用相同模式。
