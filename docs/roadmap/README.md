# Govirta 路线图

Govirta 按"北极星目标 → 单周期闭环 → 进入下一周期"的方式演进。每个周期有唯一的北极星目标、明确的承诺与范围，以及一组可勾选的完成判定项。

## 路线总览

| 周期 | 北极星 | 状态 |
| --- | --- | --- |
| [周期 1：单机单虚机闭环](./cycle-1-single-node-single-vm.md) | 用 Govirta 自有代码启动并优雅停止一台 cirros 虚机 | 进行中 |
| [周期 2：节点守护进程与多虚机生命周期](./cycle-2-node-daemon.md) | govirtlet 稳定管理本机多台虚机,崩溃可恢复 | 未开始 |
| [周期 3:控制面与调度](./cycle-3-control-plane.md) | govirtad + 多节点 + CLI 端到端调度一次完成 | 未开始 |
| [周期 4:生产最小可用集](./cycle-4-production-minimum.md) | 模板、cloud-init、VNC、扩盘、认证审计 | 未开始 |
| [周期 5:运维与弹性](./cycle-5-operations.md) | 迁移、快照、备份、监控、节点排空 | 未开始 |

## 演进原则

- **一次只服务一个北极星**:本周期不做与北极星无关的工作,范围外项目即明确推迟。
- **不留兼容层**:AGENTS.md 已规定快速迭代期不为兼容性保留技术债,周期切换时该删就删,不做过渡方案。
- **完成判定全勾选才算完成**:差一项就不算完成,不允许"基本完成"。
- **演进方向单调**:每个周期都建立在上一周期已完成的能力上,不允许回头改前一周期的承诺。
- **不接入 libvirt**:Govirta 的虚拟化栈永久只走 QEMU + QMP + qemu-img + netlink。任何周期都不评估、不规划 libvirt 接入。

## 周期状态标记规则

每份周期文档顶部维护一行状态:

- `进行中`:当前周期的工作正在推进。
- `已完成`:所有完成判定项均已勾选,并附验收证据(命令、日志、截图、PR 链接)。
- `未开始`:尚未进入该周期。

同一时间最多只有一个周期处于「进行中」。某周期标记为「已完成」后,下一周期才能从「未开始」切换为「进行中」。

## 路线图与代码的关系

- 周期文档**只描述承诺与范围**,不写技术实现。
- 具体设计写到 `docs/superpowers/specs/`,具体执行计划写到 `docs/superpowers/plans/`,两者都按周期归档。
- 路线图与 README、AGENTS.md 保持一致:三处的勾选状态必须同步更新。
